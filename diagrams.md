## Data Flow

### 1. Room Creation

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ Consumer │────>│  Chalk   │────>│Cloudflare│────>│  Chalk   │
│   App    │ 1   │   API    │ 2   │   API    │ 3   │    DB    │
│          │<────│          │<────│          │     │          │
└──────────┘  4  └──────────┘     └──────────┘     └──────────┘

1. POST /api/v1/rooms (API Key + tenant info)
2. POST /realtime/kit/{app}/meetings (CF token)
3. Store room + CF meeting_id mapping
4. Return room_id to consumer
```

### 2. Participant Join

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  Client  │────>│  Chalk   │────>│Cloudflare│────>│  Chalk   │
│   SDK    │ 1   │   API    │ 2   │   API    │ 3   │   Redis  │
│          │<────│          │<────│          │     │          │
└──────────┘  4  └──────────┘     └──────────┘     └──────────┘

1. POST /api/v1/rooms/:id/participants (JWT)
2. POST /realtime/kit/{app}/meetings/{id}/participants
3. Cache participant state in Redis
4. Return authToken for WebRTC
```

### 3. Media Streaming

```
┌──────────┐         ┌──────────────────────┐         ┌──────────┐
│  Client  │────────>│   Cloudflare SFU     │────────>│  Client  │
│    A     │ Media   │                      │  Media  │    B     │
│          │<────────│  (Anycast Routing)   │<────────│          │
└──────────┘         └──────────────────────┘         └──────────┘

• Video/Audio streams go directly to Cloudflare SFU
• SFU routes to other participants
• TURN used when direct connection fails
• Chalk backend NOT in media path (low latency)
```

### 4. Real-time Events

```
┌──────────┐         ┌──────────┐         ┌──────────┐
│  Client  │<───────>│  Chalk   │<───────>│  Client  │
│    A     │   WS    │   WS     │   WS    │    B     │
│          │         │  Server  │         │          │
└──────────┘         └────┬─────┘         └──────────┘
                          │
                          ▼
                    ┌──────────┐
                    │  Redis   │
                    │  Pub/Sub │
                    └──────────┘

Events: participant.joined, chat.message, hand.raised, etc.
```

### 5. Recording Flow

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  Client  │────>│  Chalk   │────>│Cloudflare│────>│Cloudflare│
│          │ 1   │   API    │ 2   │Recording │ 3   │    R2    │
└──────────┘     └──────────┘     │   API    │     │          │
                                  └──────────┘     └────┬─────┘
                                                        │ 4 (after 7d)
                                                        ▼
                                                  ┌──────────┐
                                                  │    S3    │
                                                  │  Glacier │
                                                  └──────────┘

1. POST /api/v1/rooms/:id/recordings/start
2. POST /realtime/kit/{app}/recordings
3. SFU captures and stores in R2
4. Lifecycle policy moves to Glacier after 7 days
```
