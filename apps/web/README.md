# Chalk web

This Vite, React 19, and TanStack Router app serves three roles:

- the public marketing shell at `/`;
- a localhost-only public-SDK consumer room at `/room`;
- a state-driven React SDK component preview at `/sdk-preview`.

It is not a complete hosted meeting product. `/new`, `/whiteboard`, `/status`, `/privacy`, and `/terms` are blank routes, and the SDK preview uses local fixtures rather than live room state. Track those gaps in the root [`checklist.md`](../../checklist.md).

Routes live in `src/routes/` and are generated by TanStack Router. Shared product behavior belongs in the SDK packages; this app should remain a thin first-party integration and verification surface.

## Local SDK room

The `/room` route creates one `ChalkSession`, renders its state through `@q9labsai/chalk-react`, and never constructs Sync or Cloudflare transports directly. Its localhost backend uses `@q9labsai/chalk-client/server` to create a temporary room and session, admit each browser tab, and refresh participant access. `/browser-session` stores only the display name behind the opaque cookie; `ChalkSession` requests camera and microphone permission before its first `/access` call creates a participant, so permission denial cannot leave an admitted participant behind.

Set only server-side environment variables before starting the app:

```bash
export CHALK_API_KEY='chalk_sk_...'
export CHALK_TENANT_ID='...'
export CHALK_API_URL='http://127.0.0.1:8080'       # optional default
export CHALK_SYNC_URL='ws://127.0.0.1:4100/v3/sync' # optional default
```

`pnpm --dir apps/web dev` binds both processes to `127.0.0.1`. The backend exposes only `/local-chalk/browser-session`, `/local-chalk/access`, and `/local-chalk/cleanup`; it stores the API key and tenant, room, session, and participant identities server-side behind an opaque `HttpOnly; SameSite=Strict` cookie. The Vite proxy forwards those paths without injecting authorization headers.

Do not point this local verification backend at production. A hosted product should implement the same narrow access-provider contract in its own authenticated backend.

## Commands

Run commands from the repository root:

```bash
pnpm --dir apps/web dev
pnpm --dir apps/web test
pnpm --dir apps/web check-types
pnpm --dir apps/web build
```
