02 / 11

Current → desired state

Replace manual wiring with a supported lifecycle.

Current: pieces leak into the app

  • A tenant bearer is exposed to browser configuration.
  • The local room invents a development Sync token.
  • Media and Sync start independently.
  • Mute and camera controls bypass Sync authorization.
  • Teardown skips the durable Leave command.
  • The default TypeScript surface exposes Effect.

Desired: the customer sees one contract

  • A server-only client holds the API key.
  • Participant access is short-lived and audience-specific.
  • ChalkSession sequences media and Sync.
  • Every control returns an observed, authorized outcome.
  • Leave commits before local resources are released.
  • React and vanilla consumers share the same runtime.

→
Chalk API Authenticates key scope. Issues independently scoped, five-minute participant credentials.
→
Customer browser Holds only one participant's Sync and media access. Refreshes through the customer backend.

Media audience

Participant-bound signaling only; never tenant-wide Room or Session authority.

08 / 11

Runtime ownership

ChalkSession is the missing composition seam.

Inputs it owns

Participant identity, access provider, device preferences, timeouts, telemetry sink, and transport factories.

State it publishes

Lifecycle, local media, participants, publications, capabilities, connection health, and privacy-safe diagnostics.

Operations it sequences

Join, reconnect, authorized media controls, moderation, role changes, hand state, display name, Leave, and end-for-all.

Join invariant

Media must be ready before Sync advertises publication intent; the Session is live only when both transports converge.

Leave invariant

Durable Leave is attempted before transports close; bounded failure still releases local devices and emits a visible diagnostic.

09 / 11

Release gate

“Publishable” means a clean consumer proves it.

Required proof

  • Pack the client and React packages from the candidate workspace.
  • Install them into an application outside the monorepo.
  • Create a Session through the server-only surface.
  • Join two real browsers with no tenant credential in either bundle.
  • Exchange audio, camera, and screen share.
  • Converge roles, hand state, moderation, reconnect, and durable Leave.

Automatic failure conditions

  • Workspace dependency leakage.
  • A browser bundle containing a tenant API key or server-only module.
  • Control success before media outcome is observed.
  • Duplicate participants after refresh or reconnect.
  • Tracks, timers, sockets, or peer connections surviving Leave.
  • An npm artifact published before the packed proof passes.
