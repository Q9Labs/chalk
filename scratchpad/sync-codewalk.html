Nine pieces of Elixir syntax

Every unfamiliar symbol in these four files is on this page. Learn these nine and the code stops being cryptic — the logic underneath is simple.

:ok   :raise_hand   :error

An atom. A constant whose name is its value, like a symbol or an enum member. Cheap to compare. Written with a leading colon.

{:ok, event, room}

A tagged tuple. Elixir doesn't throw exceptions for expected failures — it returns {:ok, …} or {:error, reason} and makes you look at which one you got.

%{"type" => "hello"}  ·  %{phase: :joined}

A map. The first has string keys (data from the wire), the second has atom keys (data we control). This app keeps that line sharp: strings outside, atoms inside.

%Room{id: "r-42", revision: 0}

A struct: a map with a fixed set of keys, declared by a module via defstruct. Typo a key and it won't compile.

def apply_event(%{revision: rev}, %{base_revision: rev})

Pattern matching in the function head. The same variable used twice means "these must be equal." Elixir picks the first clause whose patterns all match. If none match, it crashes. This is the single most important idea in the codebase.

case map do %{^key => val} -> …

The pin operator ^ means "match against the value in this variable," not "assign to it." Without the pin, key would be rebound instead of tested.

with {:ok, x} <- f(), {:ok, y} <- g(x) do … else … end

A happy-path chain. Each step must match its pattern. The first one that doesn't short-circuits — its value becomes the result (or falls to else). No nested ifs.

conn |> encode() |> send()

The pipe feeds the value on the left in as the first argument on the right. a |> f(b) is exactly f(a, b).

%{room | revision: 3}  ·  put_in(a.b[c], v)

Nothing mutates. Both of these build and return a new value. The old one is untouched and still valid. %{m | k: v} also refuses to add keys that don't exist yet.

Two more that only matter in the process layer
def foo  ·  defp foo  ·  _ignored

defp is a private function. A leading underscore on an argument means "I'm deliberately not using this" — it silences the unused-variable warning.

use GenServer  ·  @impl true  ·  @behaviour

A behaviour is an interface: a list of functions a module promises to define. use GenServer injects boilerplate; @impl true marks the functions that fulfil the contract, so the compiler catches typos.
