Shared contract freeze

Five promises must mean the same thing everywhere

This is the boundary between design and delegation. Control, capture, rendering, and qualification may split only after these identities, assignments, object grants, progress reports, and finalization rules are frozen.

Background

The recorder crosses machines, providers, storage, and delayed jobs. If each lane invents its own idea of “current worker,” “successful upload,” or “finished recording,” retries will create duplicate work or let stale machines change current state.

The control service owns intent. Workers receive the smallest temporary authority that lets one attempt succeed, then report immutable facts. Every write proves which job, attempt, fence, worker, and lease produced it.

The shared contract map

01

Identity

machine + role
Rule

A five-minute one-time bootstrap becomes a short-lived node certificate. The database records issuance, expiry, and revocation.

Failure test

A copied, expired, wrong-role, or revoked identity cannot claim or mutate work.

02

Assignment

one fenced attempt
Rule

Claim commits the job, attempt, owner, lease, deadline, and increasing fence before the assignment is returned.

Failure test

After replacement, every late heartbeat, upload report, completion, and finalization from the old fence is rejected.

03

Storage + keys

server-owned intent
Rule

The server chooses the exact object key, operation, size, condition, checksum, expiry, and encryption authority. Workers never supply arbitrary paths.

Failure test

A conflicting immutable object stops the attempt. Chalk never overwrites different bytes at an accepted key.

04

Progress + failure

lease + vocabulary
Rule

Every worker reports bounded progress, resource use, gap facts, retryability, and one terminal result using shared event shapes.

Failure test

Missed heartbeats fence and replace the attempt; duplicate or out-of-order facts stay idempotent.

05

Finalization

one transaction
Rule

One database transaction commits the video, complete audio source and chunks, exactly one transcription job per chunk, and the render result.

Failure test

A retry creates no duplicate artifact or transcription jobs, and a video cannot commit with a partial transcript handoff.

Where truth lives

Durable database

Owns decisions

Reservations, recordings, jobs, attempts, leases, fences, object intents, manifests, artifacts, cleanup, usage, and evidence.

Private object storage

Owns bytes

Encrypted capture bundles, final video, speaker map, and transcription audio chunks. It does not decide lifecycle state.

Workers

Own observations

Heartbeats, progress, resource use, verified checksums, gaps, and one fenced terminal report.

A worker never receives reusable database credentials, storage credentials, encryption keys, infrastructure credentials, or the Cloudflare application secret.

Proposed defaults to ratify

These choices close open seams without changing the product promise. They remain proposals until Hasan accepts or changes them.

1 · Usage reservation

Reserve the maximum accepted recording minutes when admission succeeds.

Recommended: simple, conservative, and atomic.

2 · Legacy mutations

Translate old recording routes into the new model, emit deprecation signals, then remove them deliberately.

Recommended: avoid an immediate breaking cutover.

3 · Worker identity

Use a dedicated recorder certificate authority whose signing key stays inside a managed key service.

Recommended: narrow trust without adding another service.

4 · Liveness

Heartbeat every 15 seconds and treat roughly 45 seconds without a valid heartbeat as lost work.

Recommended: bounded detection without noisy churn.

5 · Wire formats

Generate both the private route description and exact message schemas from shared Go types.

Recommended: one type source for HTTP and worker messages.

6 · Track ownership

The media-publication control domain owns participant-to-Cloudflare-track mapping.

Recommended: workers never infer identity from provider labels.

7 · Composition

Use GStreamer for composition, FFmpeg tools for independent verification, and pinned Inter/Noto Sans fonts.

Recommended: dynamic media pipeline plus separate proof.

8 · Machine ceilings

Ratify capture and render instance sizes only after measured headroom, deadlines, and cost evidence exist.

Recommended: evidence decides capacity, not a guessed SKU.
