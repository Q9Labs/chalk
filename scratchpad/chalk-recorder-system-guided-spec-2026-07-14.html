Background

The repository promises more than the system can deliver.

Chalk exposes recording through marketing, meeting controls, client methods, public routes, and session commands. The real custom workers only accept fixtures, and no managed recorder exists in the checked-in runtime. An old session command can reach a durable “starting” state with no adapter able to finish it.

The decision. Remove recording from Chalk-owned UI, retain SDK methods and REST routes, and make every start or mutation return recording_disabled before it can create state or cost.

Scope

Remove the product promise; retain the dormant foundation.

In scope

  • Marketing, product docs, tours, previews, and screenshots
  • Chalk web and mobile controls, defaults, diagnostics, and downloads
  • One disabled gate for retained SDK, REST, and Sync mutations
  • Detached runtime and infrastructure registration
  • Negative database, provider, and cost proof

Out of scope

  • Choosing or building another recorder
  • Removing SDK methods, REST routes, generated contracts, or schemas
  • Deleting tables, migrations, fixtures, protocols, or infrastructure source
  • Disabling live transcription or local dictation
  • Any staging or production mutation

Where work stops

The local revision passes focused API, Sync, generated-contract, web, mobile, and infrastructure proofs plus the root quality gate. A browser confirms the absent affordance. No deployment is included.

Definition of done

“Unavailable” must be observable from every side.

  • No product or documentation surface promises meeting recording.
  • Web and mobile have no recording control, state, diagnostic, or download.
  • Chalk apps explicitly configure recording false.
  • SDK methods, React/Native utilities, REST routes, OpenAPI, and generated methods remain compile-tested.
  • SDK calls return typed recording_disabled without optimistic state.
  • REST historical reads work; mutations return HTTP 409 recording_disabled with zero effects.
  • Sync schemas remain while capability and command registration are detached.
  • Legacy Sync rejects before projection, operation, job, or dispatch.
  • Historical replay remains decodable but cannot restore authority.
  • No recorder service, worker, provider bridge, reconciler, schedule, or canary is mounted.
  • Default infrastructure remains mutation-disabled at zero capacity and zero new recording spend.
  • Live transcription, uploaded-media transcription, local dictation, and ordinary media still pass.
  • API, Sync, generated-contract, root, UI, and local browser gates pass.
  • No staging or production mutation occurred.

Canonical language

Terms

Meeting recording
Server-side capture of a Chalk meeting for later playback.
Developer contract
A retained SDK method, UI utility, REST route, generated operation, or Sync schema.
Detachment seam
One explicit registration point that keeps compiled recording execution out of the active path.
Dormant foundation
Retained, compile-tested source and schema with no mounted runtime path or recurring cost.
Cost-silent
No recording-specific provider call, storage allocation, key, queue, monitor, schedule, or compute capacity.
Negative proof
An observed test showing an attempt creates no state, request, media object, or billable resource.
Re-entry gate. Recording execution returns only under a new source-of-truth spec that chooses one capture authority and proves the complete product, privacy, cost, SDK, observability, failure, and deployment state end to end. Retained contracts alone do not authorize activation.
