Background

The old plan solved a problem the provider now owns.

Chalk already has useful database and transcription foundations. It does not have an operational capture or render fleet. The refreshed launch path keeps the foundations and removes the unimplemented media machinery.

Verdict. Managed recording is the default launch architecture. The custom worker foundation stays dormant until qualification proves the managed path cannot meet a documented requirement.

PostgreSQL remains authoritative

Provider states are append-only inputs. A signed event cannot directly publish a recording, and a late event cannot move a terminal state backward.

Storage stays deliberately split

The provider can access only a dedicated ingest bucket. The canonical object lives in a separate private archive that only Chalk services can use.

Contracts

The seams implementation cannot reinterpret.

These are the few cross-lane rules worth freezing before parallel work begins.

One writer

The recording service is the only writer of public status, storage identity, and provider identity. Browser and provider payloads cannot select authority-bearing values.

Events plus reconciliation

Signed events wake work. Authoritative fetch recovers loss and resolves ordering. UUID deduplication makes replay harmless.

Fenced import

Every import compares job, attempt, owner, lease token, and fencing generation. Conditional object writes and one final transaction make retries safe.

Isolated storage

The provider sees only a dedicated ingest bucket. Canonical media and download authority stay inside Chalk.

Honest incompleteness

A missing participant track can leave video ready and transcript unavailable. Chalk records the absence; it does not invent audio or speaker identity.

Terminal means terminal

Ready, failed, and cancelled do not move backward. Late provider events remain evidence without changing the user-visible result.

Acceptance

Done means observed on one immutable release.

Unit and fixture success is necessary, not sufficient. The launch path has to survive real provider state, storage, retries, and user-visible failure.

  • Scheduled and in-meeting starts show Recording only after confirmation.
  • Host browser closure and host departure do not stop capture.
  • A verified 720p H.264/AAC MP4 downloads through Chalk authorization.
  • Participant audio maps to authenticated members or explicit unknowns.
  • Transcript work is seeded exactly once and cannot revoke video.
  • Missing, duplicate, and reordered webhooks converge safely.
  • A killed importer retries under a new fence without duplicates.
  • Wrong-meeting or invalid media is quarantined, never published.
  • Over-limit and timeout flows require explicit unrecorded continuation.
  • Twenty rooms and 100 people meet start and processing targets.
  • Ending-together recordings meet the 30-minute p95 target.
  • Cleanup removes ingest media and telemetry contains no secrets.
  • Beta status, seven-day track copy, pricing, and rollback are reviewed.
  • Production credentials and activation remain untouched.

Open decision

One product and legal choice remains.

The provider keeps participant audio-track copies for up to seven days. Composite video can upload directly to Chalk storage without a provider-managed copy.

Recommended: accept the bounded provider copy for Beta.

Name it in the data inventory and privacy review, import tracks immediately, and keep canonical retention under Chalk policy. If seven days is unacceptable, launch composite video without speaker-aware transcription. Do not rebuild custom capture merely to avoid documenting the retention choice.

Glossary

Canonical object
The verified private media object referenced by Chalk after finalization.
Composite recording
One provider-rendered video containing the meeting layout and mixed audio.
Fencing generation
A monotonic attempt number that rejects commits from expired workers.
Participant track
One participant's audio-only WebM file, mapped through provider and Chalk identity facts.
Provider fact
A signed event or authoritative API response used as input to a Chalk decision.
Recovery deadline
The bounded period for reconciliation and import retries before terminal failure.
