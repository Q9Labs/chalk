Scripts are located at: ~/.claude/skills/ios-simulator/scripts/

Key commands:
- Launch app: python scripts/app_launcher.py --launch com.q9labs.chalk-example
- Tap by text: python scripts/navigator.py --find-text "Button Text" --tap

- Screenshot: xcrun simctl io booted screenshot /path/to/output.png
- Map screen: python scripts/screen_mapper.py --verbose

Always prefix python scripts with: PATH="$HOME/bin:$PATH"

## Known Issues Discovered

1. Race condition: PreCallScreen useEffect runs before ChalkProvider initializes rtcManager
2. iOS permissions: usePermissions cannot check iOS permission status without react-native-permissions library
3. VideoView renders but shows black - stream is valid but RTCView may not display on simulator
4. CallScreen status shows "Connecting" forever in demo mode

**Status**: completed
**What I did**: Fixed race condition in useLocalStream where startStream() was called before ChalkProvider initialized rtcManager
**Decisions**: Used a `pendingStart` state to queue stream requests. When rtcManager becomes available via useEffect, pending requests are processed automatically.
**Gotchas**:

- Metro port 8081 may be occupied by old process, kill it first
- The fix queues the request instead of showing an error, preserving the expected UX
